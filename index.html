选中文字

发表于 2017-01-17

问题引入

iOS输入框中，尤其在浏览器中,点击搜索框，如果有文字的话，会默认全选所有文字，但是不弹出菜单，点击时才会弹出菜单。

解决方法

以下是选择输入框文字的方法，但方法的不足之处，是选择之后会有光标出现。但Safari效果比这个要好，Safari选择时就没有光标，点击时弹出光标和选择菜单。

//获取当前的选择
UITextRange *textRange = self.textField.selectedTextRange;
//创建结束点
UITextPosition *endPostion  = [self.textField positionFromPosition:textRange.start offset:self.textField.text.length ];
//创建新的选择区域
UITextRange *newTextRange = [self.textField textRangeFromPosition:textRange.start toPosition:endPostion];
//选择全部
[self.textField setSelectedTextRange:newTextRange];

HTTPS/SSL

发表于 2017-01-15

安全引入

目前的互联网很多使用http做的网页，是不安全的，具体体现在钓鱼网站和信息拦截篡改。目前网页多为php开发的。例如百度www.baidu.com/index.php也是可以打开的。php是输入b/s架构开发，快速开发，学习门槛低。
HTTPS: hyper text transfer protocol over secure socket layer 是http的安全版
SSL：secure socket layer 安全套接层
HTTPS = HTTP + SSL

HTTPS工作原理

  1. 客户端 先向 服务端 发送一个HTTPS请求 默认端口443端口（HTTP请求端口是80端口）
  2. 服务端获取请求后进行处理，服务端存放了一对公钥crt public和私钥crt private ，把公钥随同响应一起发送给客户端。
  3. 客户端收到响应后 验证公钥的有效性，如果验证通过，则进行后续内容，不通过显示警告信息。如果受信用，则产生一个随机密钥，并且使用这个公钥对这个随机密钥进行加密，将加密后的随机密钥发送给服务端
  4. 服务端通过私钥解密获取客户端产生的随机密钥
  5. 服务器再去把响应的内容通过这个随机密钥加密返回给客户端。
  6. 客户端使用随机密钥对响应内容进行解密

SSL协议配置

以PHP网站为例，配置SSL协议保证PHP网站的安全
证书，就是公钥。证书中有颁发者，颁发给，有效期
颁发者：CA 证书授权中心。CA是由全球受信任的审查机构去审查，其所颁发的证书是全球受信任的，CA会和操作系统厂商、浏览器厂商合作，让这些厂商把CA的根证书放到软件中。操作系统就是一个软件。通过查找证书路径，可以找到证书。
要部署SSL协议，必须需要证书，证书可以向CA申请，目前普遍的证书都是收费的,但是也存在部分的类型证书是免费的。免费的年限一般都比较短。
在部署的时候，要求证书使用的域名要和证书允许的域名一致，要么和颁发给 一致 或者 和 使用者的可选名称 中给定的域名一致。现在一般不会给ip地址颁发证书。

  1. 安装php环境,输入127.0.0.1回环地址,测试环境是否搭建成功
  2. 创建php的虚拟站点
  3. 在配置学习环境的时候，还需要去设置本地的域名解析：修改hosts文件 添加IP地址 添加域名 和第二域名 可以用ping 命令去测试域名的绑定情况 指向指定的ip就对了
  4. 编写站点中的默认Index.php文件 测试HTTPS协议访问效果。
  5. 在Apache中配置HTTPS协议环境
    配置文件： httpd-ssl.conf 引入httpd-ssl.conf
    开启ssl扩展：Loadmodule ssl_module modules/mod_ssl.so
    将证书复制到Apache文件中 位置不限 能引入就可以

    bundle.crt 证书中间信任链
    privatekey.key 私钥
    pubulickey.crt公钥

对于公钥私钥的后缀，部署时没有特殊要求，公钥一般后缀为crt，方便文件打开。
修改https-ssl.conf配置文件 修改虚拟主机配置 站点的根目录，协议站点访问域名 引用开关 加密组件 服务器证书文件 私钥配置 证书链
双向认证很少用，不用管。
重启Apache 测试ssl

有时候,你会发现,明明应用已经显示为可供销售了,但依然在App Store 无法看到已可供销售的那个版本.

问题分析及处理

查看苹果发来的邮件,
If your contracts are not in effect at this time, your app status will be Pending Contract. You may track the progress of your contracts in the Agreements, Tax, and Banking module in iTunes Connect.
To make changes to this app, sign in to iTunes Connect and open the Manage Your Applications module.
It can take up to 24 hours before your app is available on the App Store. This delay is dependent on any app availability issues.
Before you market your app, read the App Store Marketing and Advertising Guidelines for Developers. The guidelines include information on using the App Store badge, best practices to market apps on the App Store, and details on the use of Apple product images.
If you have any questions regarding your app, use the Contact Us module on iTunes Connect.
发现,

  1. 如果你的合同此时没有生效,那么你的应用将会等到合同生效后变更.可以在 iTunes Connect中的协议,税务等查看协议并更新状态.
  2. 你的应用在App Store可以使用可能需要24个小时,这根据应用存在的问题可能会有延迟.
  3. 在把app推向市场之前,阅读App Store Marketing and Advertising Guidelines 该指南包括使用应用程序商店徽章，应用程序商店的应用程序的最佳实践的信息，以及使用苹果产品图像的详细信息。
    在点击联系我们Contact Us时,跳转到联系我们的界面,选择问题主题:应用商店问题,选择下一个主题,应用不在商店时,会给出被频繁问到问题.答案就在里面.

ios位置授权一闪而过

发表于 2017-01-09

问题引入

在使用地位授权的时候，我是这么写的，（当然在plist文件中已经配置过相关键值对）
if ([UIDevice currentDevice].systemVersion.floatValue >= 8.0) {
CLLocationManager *locationManager = [[CLLocationManager alloc]init];
[locationManager requestWhenInUseAuthorization];
}

寻找原因

显示的结果是一闪而过。查看了相关文档

To configure and use a CLLocationManager object to deliver events:
1 Always request authorization to use location services and check to see whether the desired services are available as described in Requesting Permission to Use Location Services.
2 Create an instance of the CLLocationManager class and store a strong reference to it somewhere in your app.
Keeping a strong reference to the location manager object is required until all tasks involving that object are complete. Because most location manager tasks run asynchronously, storing your location manager in a local variable is insufficient.
3 Assign a custom object to the delegate property. This object must conform to the CLLocationManagerDelegate protocol.
4 Configure any additional properties relevant to the desired service.
5 Call the appropriate start method to begin the delivery of events.

第二点说了，创建一个CLLocationManager实例，并且保持一个强引用。保持强引用是因为直到定位相关的任务执行完毕，都需要这个实例。
另外给这个对象设置代理，代理必须遵守CLLocationManagerDelegate协议。

解决方案

因此，出现上述问题的原因，主要是CLLocationManager对象没有保持强引用。在代码块执行完毕就结束了，等不到用户点击进行授权。因此解决方法就是对这个对象强引用，使其成为属性或者成员变量即可。

额外信息-请求授权

The use of location services requires user authorization. In addition, some location services require the presence of specific hardware on the given device. For example, heading information is available only on devices that contain a hardware compass. Prior to using location services, your app must request authorization from the user to use those services and it must check the availability of the target services. A typical sequence for using location services is as follows:
1 Call the authorizationStatus class method to get the current authorization status for your app.
If the authorization status is kCLAuthorizationStatusRestricted or kCLAuthorizationStatusDenied, your app is not permitted to use location services and you should abort your attempt to use them.
2 Create your CLLocationManager object and assign a delegate to it.
3 Store a strong reference to your location manager somewhere in your app.
4 In iOS, if the authorization status was kCLAuthorizationStatusNotDetermined, call the requestWhenInUseAuthorization or requestAlwaysAuthorization method to request the appropriate type of authorization from the user.
5 Depending on the services you need, call one or more of the following methods:
◦ If you use the standard location service, call the locationServicesEnabled method.
◦ If you use the significant location-change service, call the significantLocationChangeMonitoringAvailable method.
◦ If you use heading information, call the headingAvailable method.
◦ If you monitor geographic or beacon regions, call the isMonitoringAvailableForClass: method.
◦ If you perform ranging on Bluetooth beacons, call the isRangingAvailable

Important
Always request authorization at the point where you actually plan to use location services to perform a task. Requesting authorization may display an alert to the user. If it is not clear to the user that your app is using location services for a useful purpose, the user may deny your request to use those services.

使用位置服务必须用户授权，而且某些功能还需要设备硬件支持，例如头部朝向需要指南针的支持，使用之前必须先请求权限，然后检查状态，一般操作如下：

  1. 调用authorizationStatus类方法 来获取应用程序当前的授权状态
  2. 创建CLLocationManager实例，并设置代理
  3. 在app的某个地方对这个实例对象保持强引用
  4. 如果授权状态是不确定kCLAuthorizationStatusNotDetermined 那么调用requestWhenInUseAuthorization请求需要时授权或者requestAlwaysAuthorization请求一直授权
  5. 根据服务需要，调用一个或多个下列方法
    • 如果使用标准的定位服务，调用locationServicesEnabled（用户可以在通用设置中关闭位置服务功能，因此要调用这个方法，判断用户定位服务是否启用，用户用户没有启动，而强行执行开始更新位置时，定位就会想它的代理发送错误通知locationManager:didFailWithError:, locationManager:monitoringDidFailForRegion:withError:）
      • 如果使用位置改变服务，调用significantLocationChangeMonitoringAvailable （是否只监听重要位置的改变，追踪用户位置，但不需要精确的位置信息）
      • 如果使用头部朝向信息，调用headingAvailable（航向是否可用，并未所有的ios设备都支持航向信息，调用此方法检查是否支持）
      • 如果监听地理位置和区域，调用isMonitoringAvailableForClass（某个类是否支持区域监听，这个类必须继承CLRegion）:
      • 是否支持蓝牙信标测距，调用isRangingAvailable

遗留问题

想在程序启动时就进行位置授权请求，因此在app代理中，当程序启动时请求授权，代理对CLLocationManager强引用，没设置代理，直接请求授权。但在程序使用位置信息时，切换到后台，苹果会在状态栏显示该应用正在后台使用位置信息。分析的可能原因，就是没有再合适的时候，将强引用的这个对象释放掉，导致程序使用位置时指向的是这个定位管理者，而定位管理者没有在合适的时候释放。
尝试的解决方案就是让应用代理本身作为CLLocationManager的代理，直接请求更新位置，实现其代理方法，成功或失败，都停止更新，并且除去强引用。不知道会不会引起其他问题。不行的话就单独创建一个类，这个类对象去做刚才应用代理做的那些事情，完成之后，将其置为nil。
尝试代码：
(BOOL)application:(UIApplication )application didFinishLaunchingWithOptions:(NSDictionary )launchOptions {
// Override point for customization after application launch.
if ([UIDevice currentDevice].systemVersion.floatValue >= 8.0) {
if (![CLLocationManager locationServicesEnabled]) {
NSLog(@”请先开启定位服务”);
return YES;
}
self.locationManager= [[CLLocationManager alloc]init];
self.locationManager.delegate = self;
[self.locationManager startUpdatingLocation];
[self.locationManager requestWhenInUseAuthorization];
}
return YES;
}
-(void)locationManager:(CLLocationManager )manager didUpdateLocations:(NSArray<CLLocation > )locations{
[self.locationManager stopUpdatingLocation];
self.locationManager = nil;
self.locationManager.delegate = nil;
}
-(void)locationManager:(CLLocationManager )manager didFailWithError:(NSError *)error{
[self.locationManager stopUpdatingLocation];
self.locationManager = nil;
self.locationManager.delegate = nil;
}

2016年下半年，应该是我最艰难的时候，也是我最为孤独的时候。

　　从事的工作，整天对着电脑，有时候，会让我十分厌烦电子产品，因为我觉得我的眼睛受到的电子产品的刺激，去年的时候眼睛还没这么累，今年在深圳，工作一段时间，总觉得眼睛不舒服，眼睛干涩难受。所以，有时候我一点都不想摸电脑，不想玩手机。电脑可以不玩，但手机还是少不了瞅两眼，尤其是我在如此孤单的情况下，手机就是我经常的陪伴。
我现在还单身，可能是不知道抓住机遇，也可能是缘分未到，刚出道的这几年，我的心思其实更多的在工作上，在事业上，想站住脚跟，所以对象这档子事，有时候也蛮分心的，有时候觉得两样都没搞好，事业不如想象中的那么牛逼，而自己还是个单身狗。
　　今天下午，眼睛实在干涩难受，在屋里呆了一天，实在不想待在屋里，于是我就决定一个人去散散步。去哪里呢？想起来以前来深圳玩的时候，波哥带我去过深圳北站附近的一个公园，我挺喜欢那里的。那是繁华都市，高楼林立丛中的一片净土，环境很好，我摸索着道路去了，满天的风筝飞舞，草坪上有很多亲子活动，也有好多的情侣，还有一些背着吉他在这里卖唱的。而我一个人来这里散步，没错，我就是来这里走走，看看悠闲的人们，看看蓝天，听听那些背着吉他追寻梦想的人的歌声。我躺在草坪上，看看高高的蓝天，心情平静多了，心里想着，要是有个人陪我就好了。
　　我在想，如果我谈了对象，我希望我们能生活在一起，而不是住的不在的城市，我不喜欢异地恋，尤其是遥远的异地恋。因为，我从未有过如此强烈的孤独感。一个人吃饭，一个人散步，一个人过自己的生活。

过年回家，一票难求

　　快过年回家了，我的火车票依然没抢到，我的内心还是有些不安的，毕业后没抢过票，没想到形势如此严峻，尤其是在深圳，车票一出来就被秒光了。我现在用了n多软件在抢票，抢了半个多月了吧，杳无音讯。我甚至开始痛恨黄牛，回家的火车票的确是个刚需，但拿这个去赚钱是不是太不道德了。我希望苍天保佑我能买到回家的票和返程的票，希望那些黄牛不要做伤天害理的事，还老百姓一个普普通通的回家的车票。

2017 期待遇见更好的自己

　　2017元旦的时候，大家都在说自己未来的计划，我不知道自己未来的计划是什么，也从来不想给自己列什么计划，因为我之前太爱给自己列计划了，然后很多事情总是计划赶不上变化，于是我习惯了走一步算一步的生活。但我并不堕落，不列计划，但是心里会有一个大致的方向，尽管朝着那方方向走就对了，至于能走多远，就看自己在时间长河中的付出了。
我知道成长需要养成良好的习惯，可对我来说，习惯总是那么的难以养成，比如养成造成的习惯。比如养成每天写博客的习惯。比如每天锻炼的习惯。好的习惯会让我们变成更好的自己。甚至是决定一个人成功失败的重要原因，不可小觑习惯的力量。
　　尽管习惯难养成，但尽量做到最好吧。我习惯2017年能养成几个终生受益的好习惯，比如锻炼身体，比如看书学习，比如坚持写点什么。
2017年，期待遇见更好的自己。遇见更好的你。
